## 내용정리 ##
1. 자식 클래스의 생성자가 호출되면, 자동으로 부모 클래스의 매개 변수 없는 생성자가 실행된다.

2. 자식 클래스에서는 부모클래스에 있는 public, protected로 선언된 모든 인스턴스 및
클래스 변수와 메소드를 사용할 수 있다.

3. 다중 상속은 불가능 하다. 오직 단일 상속만 가능하다.

4. 자식클래스에서 매개변수가 있는 생성자를 호출하여도 부모의 매개변수가 없는 기본생성자를 찾는다.
그래서, 부모클래스에 매개변수가 있는 생성자만 있을 경우 super()를 이용해서 부모 생성자를 꼭
호출해야 한다.

5. 자식클래스의 생성자에서 super()를 명시적으로 지정하지 않으면, 컴파일시 자동으로 super()
가 추가된다. 또한 super()는 반드시 자식클래스의 생성자 가장 첫줄에 선언되어야 한다.

6. 메소드 overriding은 접근제어자, 리턴타입, 메소드이름, 매개변수타입이 일치해야한다.

7. 상속을 할때 접근제어자가 확대되는 것은 상관없지만 축소되는것은 안된다.

8. instanceof를 사용하여 타입을 점검할때에는 가장 하위에 있는 자식타입부터 확인해야한다.

9. 자식 타입의 객체를 부모타입으로 형변환하는 것은 자동으로 된다.

10. 부모타입의 객체를 자식 타입으로 형변환할때에는 명시적으로 타입을 지정해주어야 한다.
이때, 부모타입의 실제 객체는 자식타입이어야 한다.

11. instanceof로 타입 확인을 할 때 부모 타입도 true라는 결과를 제공한다.

12. 형변환을 하더라도 실제 호출되는 것은 원래 객체에 있는 메소드가 호출된다. => 다형성


## QUIZ ##
1. 상속을 받은 클래스의 선언문에 사용하는 키워드는 무엇인가요?
- extends

2. 상속을 받은 클래스의 생성자를 수행하면 부모의 생성자도 자동으로 수행되나요?
- O

3. 부모클래스의 생성자를 자식 클래스에서 직접 선택하려고 할때 사용하는 예약어는 무엇인가요?
- super

4. 메소드 Overriding과 Overloading을 정확하게 설명해 보세요.
- Overriding : 상위 클래스가 가지고 있는 메소드를 하위클래스가 재정의
- Overloading : 같은 이름의 메소드 여러개를 가지면서 매개변수의 타입과 개수가 다르다.

5. A가 부모, B가 자식 클래스라면 A a = new B();의 형태로 객체 생성이 가능한가요?
- O

6. 명시적으로 형변환을 하기전에 타입을 확인하려면 어떤 예약어를 사용해야 하나요?
- instanceof

7. 6에서 사용한 예약어의 좌측에는 어떤 값이, 우측에는 어떤값이 들어가나요?
- 좌측에는 객체(참조변수명), 우측에는 클래스(타입)명

8. 6의 예약어의 수행 결과는 어떤 타입으로 제공되나요?
- boolean

9. Polymorphism이라는 것은 도대체 뭔가요?
- 하나의 객체가 여러가지 타입을 가질수 있는 것을 의미한다. 부모클래스 타입의 참조 변수로
자식클래스 타입의 인스턴스를 참조할 수 있도록 한다.
